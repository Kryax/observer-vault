# Instrument-Before-Product

## Structural Description

A system builds the tools for observing and evaluating its domain before it builds the thing that operates in that domain. The instruments are not scaffolding to be discarded after use — they become permanent infrastructure that the product uses at runtime. The observation tools and the product share the same substrate, so building the instruments first establishes a foundation that constrains and informs the product's architecture.

## Domain-Independent Formulation

Systems that build their observational and evaluative instruments before their primary product produce products whose architecture is constrained by what the instruments can see.

## Instances

### Instance 1: Observer Project Development
- **Domain:** Software Project Development
- **Expression:** The Observer vault (an observation/classification system for documents) was built before any project it now contains. The OCP scraper (an observation instrument for external codebases) was built before the motif system needed external validation. The experiment apparatus (E1, E3) was built before the council architecture was finalised. Session memory (S2) was built as infrastructure before the cognitive loop (SKILL.md) that references it. In each case, the instrument became permanent runtime infrastructure, not discarded scaffolding.

## Discovery Context

Identified during an archaeological triad analysis of the Observer project's own development history on 9 March 2026. The convergence of all three independent perspectives (Materials, Decision, Tempo) revealed that observational infrastructure consistently preceded product infrastructure across every phase of the project — from the vault's founding governance documents through to session memory preceding the cognitive loop.

## Falsification Conditions

- If the instruments built early are demonstrably unused by the later product (they were scaffolding that happened to survive, not load-bearing infrastructure), then this is survivorship bias rather than a structural pattern
- If systems that build product-first and retrofit observational instruments achieve equivalent quality and architectural coherence, then the build order is incidental rather than structurally meaningful
- If the pattern only holds when the product IS an observation system (circular case), and does not generalise to products in other domains, then it describes a tautology rather than a transferable motif
